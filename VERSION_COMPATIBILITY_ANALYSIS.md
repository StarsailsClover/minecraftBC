## 🛠️ 添加版本支持的方案

### 方案1: 快速添加 (推荐)

直接向 `ProtocolVersion` 枚举和 `PACKET_IDS` 映射添加新版本:

```python
class ProtocolVersion(IntEnum):
    # 现有
    V1_12_2 = 340
    V1_16_5 = 754
    V1_18_2 = 758
    V1_19_2 = 760
    V1_20_1 = 763
    V1_20_4 = 765
    
    # 新增
    V1_8_9 = 47       # 如果需要
    V1_17_1 = 755
    V1_18_1 = 756
    V1_19_1 = 759
    V1_20_6 = 766
    V26_1 = 770       # 待确认
```

### 方案2: 从Jar文件提取协议

对于26.1等未知版本，可以从jar文件提取:

```python
# 从version.json提取
import json

with open('versions/26.1/26.1.json') as f:
    data = json.load(f)
    protocol = data['protocolVersion']
```

### 方案3: 简化支持 (更务实)

大多数相邻版本的包ID相同或相似，可以:

1. 为每个版本创建独立适配器
2. 使用 `for_version()` 动态创建
3. 对于未知版本，尝试使用最接近的已知版本
